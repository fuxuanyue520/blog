我们在日常工作中会遇到各种各样的问题，解决问题是能力，但**如何从解决问题的过程中提炼出通用的知识**，才是成长的关键。

## 知识的三个层次

1. **点**：解决具体的 Bug，学会某个 API 的用法。
2. **线**：串联起相关的知识点，比如理解了 React 的生命周期，从而理解了 Hooks 的设计动机。
3. **面**：构建完整的知识体系，比如前端工程化、后端微服务架构。

## 实践建议

- **复盘**：项目结束后，花时间回顾遇到的坑和解决方案。
- **输出**：通过博客、分享会等形式，倒逼自己整理输出。
- **重构**：不仅是代码重构，也是认知重构。随着经验增长，定期更新自己的知识库。

## 总结

建立知识体系是一个漫长的过程，不要急于求成。保持好奇心，持续学习，时间会给你答案。
